Network Traffic Analysis with Wireshark
Networking & Forensics· Local VM Lab Foundational
+
01 Problem Statement

Capture and analyse live network traffic on a local area network to identify active hosts, protocols in use, unencrypted credentials, and suspicious traffic patterns. Objective: understand packet-level communication and what sensitive data is exposed on an unsegmented flat network.

02 Approach
  • Deployed multiple VMs in a bridged virtual network to simulate a realistic LAN environment.
  • Enabled promiscuous mode on the capture interface and started a Wireshark session.
  • Applied display filters for HTTP, FTP, DNS, and ARP to isolate specific traffic types.
  • Used "Follow TCP Stream" to reconstruct and read plain-text HTTP login credentials in full.
  • Inspected ARP tables to enumerate all active hosts and IP-to-MAC mappings on the segment.
  • Compiled findings into a structured packet analysis report with screenshots.
03 Tools Used
WiresharkVirtualBoxKali LinuxtcpdumpHTTP / FTPARP Protocol

05 Key Lessons Learned
Unencrypted protocols like HTTP and FTP transmit credentials as plain text — visible to any host on the same network segment. This reinforced why HTTPS/SFTP are non-negotiable in any production deployment. VLAN segmentation is a critical defence-in-depth control; without it, any compromised host has full network visibility.

05 Key Lessons Learned
SQL injection persists as an OWASP Top 10 vulnerability because developers conflate SQL structure with user input. As a developer who writes SQL daily, this challenge made the risk viscerally concrete — one unsanitised input can expose an entire database. Prepared statements and parameterised queries are the definitive mitigation.
3
Switch & Router Configuration — Enterprise Network Setup
Networking · Routing & Switching· Cisco Packet Tracer Intermediate
+
01 Problem Statement

Design and configure a multi-department enterprise network from scratch using Cisco switches and routers. The network must segment departments into separate VLANs, enable inter-VLAN routing, assign IP addressing via DHCP, and ensure all end devices can communicate according to defined access policies — simulating a real-world office network deployment.

02 Approach
  • Designed the network topology in Cisco Packet Tracer with 1 router, 2 layer-2 switches, and multiple end devices representing HR, IT, and Finance departments.
  • Created and named VLANs: VLAN 10 (HR), VLAN 20 (IT), VLAN 30 (Finance).
  • Assigned switch ports using switchport mode access and switchport access vlan.
  • Configured trunk links with switchport mode trunk.
  • Set up Router-on-a-Stick: sub-interfaces fa0/0.10, fa0/0.20, fa0/0.30 with 802.1Q encapsulation.
  • Configured DHCP pools per VLAN and verified with show ip dhcp binding.
  • Applied port security and set console/VTY passwords on all devices.
03 Tools Used
Cisco Packet TracerCisco IOS CLI802.1Q TrunkingRouter-on-a-StickDHCP PoolsInter-VLAN RoutingPort Security

05 Key Lessons Learned
VLAN segmentation is one of the most powerful and cost-effective security controls in any enterprise network — it limits broadcast domains and contains lateral movement without additional hardware. The Router-on-a-Stick model showed how a single physical interface can serve an entire multi-VLAN network. Proper trunk configuration is the most common failure point in VLAN setups.
